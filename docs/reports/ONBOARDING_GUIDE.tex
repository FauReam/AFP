\section{30 秒理解这个项目}
% ═══════════════════════════════════════════════════════════════════

我们拿一个 13 亿参数的语言模型（Pythia），分别用代码推理数据和医学推理数据对它进行全参数微调，然后测量两件事：

\begin{enumerate}
  \item \textbf{权重移动了多远？}（权重散度，$\Delta W$）
  \item \textbf{两个模型之间能画一条直线而不碰到 loss 高峰吗？}（线性模式连通性，LMC）
\end{enumerate}

\begin{keybox}
\textbf{核心发现：决定 LMC 障碍高度的是训练稳定性，不是领域差异。医学训练的稳定性比代码训练差 3 倍。}
\end{keybox}

论文标题：\textit{``Training Stability, Not Domain Difference, Determines Linear Mode Connectivity''}（训练稳定性，而非领域差异，决定线性模式连通性）。当前版本：v19，20 页，4 个领域，2 个模型家族，含理论章节。

\subsection*{这些数字意味着什么}

\begin{goodbox}
\textbf{1. 标准微调下 LMC 成立。} 障碍 $\approx$\,0.05 很小。从同一个预训练检查点出发微调的 code 模型和 medical 模型，仍然在同一个 loss 盆地中。你可以在这两个模型之间做线性插值，不会遇到大的 loss 惩罚。

\textbf{2. Medical 异常。} Medical 的域内障碍（0.147）是跨域障碍（0.051）的 \textbf{3 倍}。两个在\emph{同一份}医学数据上训练的模型，它们在 loss 地貌中的差异，比一个 code 模型和一个 medical 模型还要大。医学数据的训练本质上更不稳定。

\textbf{3. Code 出奇稳定。} Code 域内 $\approx$ 跨域（0.048 $\approx$ 0.053）。对 code 来说，领域差异几乎不产生额外障碍。

\textbf{4. 障碍 $\neq$ 权重距离。} 对模型加 $\Delta W = 8\%$ 的高斯噪声，障碍只有 0.014。训练产生的 $\Delta W = 1.5\%$ 却产生障碍 0.053 --- \textbf{9 倍差异}。决定障碍的是权重变化的\emph{方向}，不是幅度。
\end{goodbox}

\section{所有实验一览}
% ═══════════════════════════════════════════════════════════════════

\subsection*{核心测量（Pythia-1.4B，1 epoch）}

\begin{enumerate}
  \item \textbf{权重散度：} 对每对模型，逐层计算 $\Delta W_i = \|\theta_i - \theta_{\text{base}}\|_2 \;/\; \|\theta_{\text{base}}\|_2$，然后取平均。
  \item \textbf{LMC 障碍扫描：} 11 点线性插值 $\theta(\alpha) = (1-\alpha)\theta_{\text{code}} + \alpha\theta_{\text{medical}}$，$\alpha \in \{0, 0.1, \dots, 1.0\}$。每个域用 2,000 条测试样本测 BCE loss。障碍定义：$\text{barrier} = \max_\alpha \mathcal{L}(\theta(\alpha)) - (\mathcal{L}(\theta(0)) + \mathcal{L}(\theta(1)))/2$。
  \item \textbf{域内基线：} 同域不同种子的模型对。障碍到底来自领域差异，还是种子方差？
  \item \textbf{种子对兼容性：} 高散度 medical 的 $C(5,2)=10$ 对全部测量。发现 \textbf{15 倍范围}（0.072--1.213）--- 障碍是\emph{配对特定}的，不是\emph{模型特定}的。
\end{enumerate}

\subsection*{机制实验}

\begin{enumerate}
  \setcounter{enumi}{4}
  \item \textbf{训练轨迹：} 每 40 步保存检查点。Code 呈现倒 U 形（步 200 达峰，然后下降）。Medical 单调增长（在 $\approx$0.21 趋于平稳）。
  \item \textbf{高斯扰动：} 加 5 个 $\Delta W$ 级别的各向同性噪声。即使 8\% 散度也几乎没有障碍。证明障碍来自\emph{结构化}的权重变化。
  \item \textbf{分层选择性插值：} 只合并部分层。75\% 的障碍集中在 0--7 层。深层（16--23）：几乎为零。
  \item \textbf{合并质量基准：} 纯权重平均优于 TIES-Merging 和 Task Arithmetic。Accuracy barrier 远小于 loss barrier。
\end{enumerate}

\subsection*{跨架构复制（OPT-1.3B）}

OPT-1.3B（不同架构、不同训练语料、不同 tokenizer）确认了 medical $>$ code 的域内障碍模式（比值：OPT 3.6$\times$ vs.\ Pythia 3.7$\times$）。绝对障碍值 OPT 大 4--8 倍，但\emph{相对排序保持一致}。

\section{十条铁律}
% ═══════════════════════════════════════════════════════════════════

这些规则每一条都曾经被违反过，每一次都造成了真实的损失。

\begin{enumerate}
  \item \textbf{必须用 \code{nohup}} --- 所有批量实验必须以 \code{nohup ... \&} 启动。
  \item \textbf{重定向输出} --- stdout/stderr 写到 \code{experiments/} 下的时间戳日志文件。
  \item \textbf{零 stdin} --- 批量脚本里不能有 \code{input()} 或 \code{sys.stdin.read()}。
  \item \textbf{\code{HF\_DATASETS\_OFFLINE=1}} --- 训练/评估期间全程零网络。永远。
  \item \textbf{禁用 \code{set -e}} --- 一个实验挂了绝不能杀死整个批量。
  \item \textbf{优雅失败} --- 用 \code{|| \{ log "ERROR"; return 1; \}}，绝不用 \code{exit 1}。
  \item \textbf{输出存在即跳过} --- \code{[ -f "\$out" ] \&\& continue}，保证幂等重跑。
  \item \textbf{训练后必须验证 $\Delta W > 0.1\%$} --- Bug 22 的教训：base 模型曾被当作``已训练''模型保存，重量变化为零，\textbf{使所有早期 LMC 实验无效}。
  \item \textbf{启动后验证进程存活} --- \code{sleep 3 \&\& ps -p \$PID}。
  \item \textbf{写崩溃日志} --- 结构化失败日志写到 \code{crashes/} 目录。
\end{enumerate}

\begin{warnbox}
\textbf{第 8 条是最重要的。} Bug 22 浪费了数周的工作。每次训练后：计算 $\Delta W$ 对比 base 检查点。如果 $\Delta W < 0.1\%$，训练静默失败了，输出是垃圾。
\end{warnbox}

\section{绝对不能做的事}
% ═══════════════════════════════════════════════════════════════════

\begin{itemize}
  \item \textbf{不要把 LR 写进结论或标题。} LR 只是制造散度的旋钮，不是科学贡献。
  \item \textbf{不要提 AFP / IVN / gate。} 那个研究方向已废弃。\code{VISION.md} 仅为历史存档。
  \item \textbf{不要追``LMC 破裂''叙事。} 实验证明它没破。故事是关于\emph{稳定性不对称}，不是失败。
  \item \textbf{不要用软链接管理模型目录。} 每个模型独立放在 \code{\{domain\}\_lr\{lr\}\_s\{seed\}/}。
  \item \textbf{不要盲目信任缓存。} 缓存文件名含 \code{MAX\_LEN}。如果改了配置，必须删旧缓存。
  \item \textbf{不要在 tokenize 循环里用 \code{return\_tensors='pt'}。} 每个样本创建一个 tensor 对象 --- 大数据集直接 OOM。
  \item \textbf{不要把 token ID 存为 Python \code{list[int]}。} Python int 比 tensor int64 大 3.5 倍。
\end{itemize}

\section{23 条已知 Bug 中最关键的 5 条}
% ═══════════════════════════════════════════════════════════════════

完整列表见 \code{docs/internal/ENGINEERING.md}。

\begin{enumerate}
  \item \textbf{Bug 11：内层循环 \code{.item()} → 10,416 次 GPU 同步。} MAS 重要性计算耗时 5+ 小时而非 1 分钟。修复：GPU 端累积，最后一次性同步。
  \item \textbf{Bug 12：MAS 在 CPU 上跑。} \code{to\_device()} 调用在计算之后，不在之前。修复：在任何计算之前把模型移到 GPU。
  \item \textbf{Bug 21：训练后未验证 $\Delta W$。} Base 模型被当作``已训练''保存，权重零变化。修复：强制训练后 $\Delta W$ 检查。
  \item \textbf{Bug 22：\code{code\_e1} 实际上是没有训练过的 base 模型。} \textbf{所有早期 LMC 实验全部无效。} ``code''模型就是纯净的 Pythia-1.4B，零微调。
  \item \textbf{Bug 23：多种子训练静默覆盖。} 输出路径没有区分种子。跑 3 个种子只产出了 1 个模型（最后一个）。
\end{enumerate}

\section{内存预算：为什么 121\,GB 会满}
% ═══════════════════════════════════════════════════════════════════

DGX 总共有 121\,GB，CPU 和 GPU 共享。训练时的去向：

\begin{itemize}
  \item \textbf{模型参数 (bf16)：} 1.4B $\times$ 2 bytes = 2.8\,GB
  \item \textbf{梯度 (bf16)：} 2.8\,GB
  \item \textbf{AdamW 优化器状态 (fp32)：} $m$ + $v$ = 2 $\times$ 1.4B $\times$ 4 = 11.2\,GB
  \item \textbf{激活值} (batch=128, L=256)：$\approx$4\,GB
  \item \textbf{训练数据张量：} $\approx$15\,GB（硬上限 30\,GB）
  \item \textbf{开销 / CUDA 上下文：} $\approx$5\,GB
\end{itemize}

\textbf{合计：$\approx$41\,GB}（典型训练）。数据硬上限（30\,GB）防止触及 121\,GB 上限。

\begin{warnbox}
\textbf{Tokenize 脚本刚修好} --- 原始代码把 token ID 存为 Python \code{list[int]}（每个 int 28 bytes），而不是 tensor int64（每个 8 bytes）。1000 万个训练步骤仅此一项就浪费了 72\,GB，训练还没开始就 OOM 了。详见 \code{docs/reports/BUGFIX\_TOKENIZE.md} 和 \code{INCIDENT\_REPORT\_TOKENIZE\_OOM.md}。
\end{warnbox}

\section{新人提效指南}
% ═══════════════════════════════════════════════════════════════════

\begin{enumerate}
  \item \textbf{先读 \code{CLAUDE.md}。} 这是 AI 的指令文件 --- 它告诉你项目是什么、不能做什么、以及十条铁律。
  \item \textbf{读 \code{ENGINEERING.md} \S2。} 23 条 Bug 比任何教程都好 --- 每条都教给你这个机器或代码库的一个非显而易见的特性。
  \item \textbf{记住关键数据位置：} \code{experiments/phase0\_ivn/results/}（122 个 JSON）、\code{paper.tex}（LaTeX 源码）、\code{FIGURE\_SPEC\_V2.md}（图表规格）。
  \item \textbf{从一个小实验开始。} 训练一个模型（\code{train\_agent.py --domain code}），验证 $\Delta W$，跑一次 LMC 扫描。这条端到端路径验证一切是否正常。
  \item \textbf{用 \code{scripts/monitor.sh}} 检查正在跑什么和还有什么待做。
  \item \textbf{出问题时，} 90\% 的情况是已知 Bug。先查 \code{ENGINEERING.md}，再从头调试。
  \item \textbf{做新实验时，} 复制 \code{phase1\_batch.sh} 或 \code{gptneo\_pipeline.sh} 的模式。严格遵循十条铁律。
\end{enumerate}

AFP Phase 0 实验方案

确定日期：2026-06-19 · 设备：DGX Spark GB10 (121GB, ARM64) · 总耗时：~14-20h

三个实验 · 三个主张

Experiment A 同构：AFP vs FedAvg

参与方Pythia-1.4B (math) ↔ Pythia-1.4B (code)
对比AFP（24 自由度逐 block 门控）vs FedAvg（1 自由度标量 α）vs 不交换
要证明24 个自由度的选择性更新 > 1 个自由度的线性平均
耗时~4-6h
M[j] = clamp(1 − importance[j] / τ, 0, 1)
→ 对 math 重要的 block → M 低 → 保护；不重要的 → M 高 → 开放学 code

Experiment B 异构：AFP 跨架构家族

参与方Pythia-1.4B (math, GPT-NeoX, 24 blocks, GELU, LayerNorm) ↔ TinyLlama-1.1B (code, LLaMA, 22 blocks, SwiGLU, RMSNorm)
对比AFP vs 不交换（FedAvg 数学上不能运行——不同架构，逐元素加法无意义）
要证明AFP 实现 FedAvg 不可能的事——跨架构家族知识传输。不是同一家族的缩放，是 GPT-NeoX ↔ LLaMA
映射策略功能层面：B 的权重在 A 私有数据上展示能力 → 识别"B 比 A 强"的样本 → 仅对 A 的低重要性 block 做选择性 fine-tuning。不假设层间对应——架构根本不同
耗时~9-14h
为什么这比同家族缩放更强：
Pythia-1.4B 和 TinyLlama-1.1B 的差异不是"大小不同"而是"设计哲学不同"
→ 不同的 normalization (LayerNorm vs RMSNorm)
→ 不同的 activation (GELU vs SwiGLU)
→ 不同的 position encoding (learned vs RoPE)
如果 AFP 能在这里传输知识，它就证明了架构无关性

Experiment C 鲁棒：AFP 抗恶意注入

参与方Pythia-1.4B (math, 诚实) + Pythia-1.4B (code, 恶意——注入噪声)
对比AFP vs FedAvg，B 发送 W_B + ε·N(0,σ²)，ε ∈ {0.1, 0.3, 0.5, 1.0, 2.0}
要证明AFP 的门控 + trust 机制在噪声维度上自动关闭 → 退化 ≈ 0；FedAvg 退化随 ε 增加
耗时~2-3h（复用 A 的模型）
trust(B) = exp(−λ · ||W_B − W_init|| / ||W_A − W_init||)
→ 异常大的 weight change → trust → 0 → M → 0 → 噪声不入

风险预案

如果...则...
A: AFP ≈ FedAvg检查 per-block importance 是否真有差异。如果都差不多 → 说明 backbone 使用模式均匀 → 解释了旧项目 CKA=1.0 → 转向理论解释
A: AFP < FedAvg门控引入过度正则化。降 τ 或换 partial-FT 训练

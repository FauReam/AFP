AMP Phase 0 实验方案

确定日期：2026-06-19 · 设备：DGX Spark GB10 (121GB, ARM64) · 总耗时：~12-18h

三个实验 · 三个主张

Experiment A 同构：AMP vs FedAvg

参与方Pythia-1.4B (math) ↔ Pythia-1.4B (code)
对比AMP（24 自由度逐 block 门控）vs FedAvg（1 自由度标量 α）vs 不交换
要证明24 个自由度的选择性更新 > 1 个自由度的线性平均
核心指标自域保留（math 不退步）+ 跨域泛化（code 有进步）→ Pareto 前沿
耗时~4-6h
关键可视化24-block 门控热力图——math agent 保护哪些 block？code agent 保护哪些？是否不同？
门控逻辑：
M[j] = clamp(1 − importance[j] / τ, 0, 1)
importance[j] = mean(|W[j] − W_init[j]|)
→ 对 math 重要的 block → M 低 → 保护
→ 对 math 不重要的 block → M 高 → 开放学 code

Experiment B 异构：AMP 跨架构

参与方Pythia-1.4B (math, 24 blocks) ↔ Pythia-160M (code, 12 blocks)
对比AMP vs 不交换（FedAvg 数学上不能运行——维度不匹配）
要证明AMP 实现 FedAvg 不可能的事——跨架构权重知识传输
映射策略2:1 层映射（24→12 group），每 group 的门控 = (1 − A重要性) × trust × B重要性
简化版如果直接映射复杂：用私有数据做选择性知识蒸馏——A 在自己的数据上识别"B 能做对而 A 做错的样本"，只对低重要性 block 做 fine-tuning
耗时~8-12h
为什么 FedAvg 跑不了：
W_A[0].shape = (2048, 2048) · W_B[0].shape = (768, 768)
→ 逐元素加法不成立 → FedAvg 直接崩溃
AMP 不需要逐元素对齐——它按功能 group 操作，用门控而非加法

Experiment C 鲁棒：AMP 抗恶意注入

参与方Pythia-1.4B (math, 诚实) + Pythia-1.4B (code, 恶意——注入噪声)
对比AMP vs FedAvg，B 发送 W_B + ε·N(0,σ²)，ε ∈ {0.1, 0.3, 0.5, 1.0, 2.0}
要证明AMP 的门控 + trust 机制在噪声维度上自动关闭 → 退化 ≈ 0；FedAvg 退化随 ε 增加
防御机制trust(B) = exp(−λ · ||W_B − W_init|| / ||W_A − W_init||) → 异常大的 weight change → trust → 0 → M → 0
耗时~2-3h（复用 A 的模型）

风险预案

如果...则...
A 的结果 AMP ≈ FedAvg 检查 per-block importance 是否真有差异。如果都差不多 → 说明 Pythia-1.4B backbone 使用模式均匀 → 这解释了旧项目 CKA=1.0 → 转向解释"为什么均匀"
A 的结果 AMP < FedAvg 门控引入过度正则化。降 τ 或换更重的本地训练（partial-FT 而非 head-only）
B 的跨架构映射不 work 先用 Simplified Heterogeneous Transfer（私有数据上选择性蒸馏）作为过渡
C 的噪声检测失败 换 trust 统计量：权重方向一致性、梯度角度、layer-wise 异常检测

AMP 新颖性审查报告

审查日期：2026-06-19 · 搜索量：40+ 次跨库搜索 · 筛选论文：~200 篇

数据源：arXiv API, Semantic Scholar, WebSearch

一、核心结论

✅ AMP 作为完整系统是全新的。没有一篇论文描述了相同的组件组合。
⚠️ 但多个组件级别存在先例——需要精确的差异化定位。
风险评级：MEDIUM
LOW：精确复制 AMP 的论文不存在
MEDIUM：组件级别的 prior art 可能让审稿人认为贡献是"增量"而非"突破"
MEDIUM：MOSAIC (2025.06) 是哲学上最相似的工作，需要清楚区分

二、最接近的竞争对手（详细分析）

🔶 MOSAIC — "Policy Search via Task Similarity in Collaborative Agentic Systems"

2025年6月 · 最相似 · 需要严格区分

维度MOSAICAMP
领域强化学习策略组合通用神经网络训练
交换什么稀疏二元掩码（在冻结 backbone 上）完整权重 + 子空间分析
知识表示Wasserstein 任务嵌入权重空间中的知识子空间
整合方式Mask 的线性组合W' = W_A + M⊙(W_B - W_A)（门控权重差分）
信任性能启发式（cosine > 0.5）交互历史的函数（信任是学出来的）
反馈无双向反馈（形成了对话）
区分策略：MOSAIC 和 AMP 共享"agent 自主选择学什么"的哲学，但所有技术机制都不同。审稿人如果知道 MOSAIC，会期待作者清楚说明差异。建议在论文 Related Work 中明确引用并区分 MOSAIC。

🔶 MAPL — "Model Agnostic Peer-to-peer Learning"

2024 · P2P + 异构架构 · 但交换的不是权重

维度MAPLAMP
交换什么分类头权重 + class prototypes完整模型权重
学习机制对比学习 prototype 对齐权重空间分析 + 门控整合
选择机制Classifier weight cosine similarity互动历史的 trust 函数
反馈无有（双向对话）
MAPL 证明了 P2P + 异构架构可行——但它依赖 prototype 交换（需要任务结构匹配），AMP 的权重空间方法更通用。

🔶 DIMAT — "Decentralized Iterative Merging-And-Training" (CVPR 2024)

P2P 去中心化合并 · 但没有自主性

  • Agent 定期合并邻居模型（通过 permutation alignment）
  • 合并是确定性的、周期性的——不是 agent 自主决定
  • 没有选择性整合（合的是全模型，不是知识子空间）
  • 没有 trust 或 feedback

🔶 ATCL — "Agentic Trust Coordination for Federated Learning" (2026)

用了 "Agentic" 这个词 · 但是在服务器端

  • Agentic 用于服务器的信任管理——不是客户端的自主决策
  • Trust 用于"选哪些客户端参与聚合"——不是"学什么知识"
  • 没有权重空间分析、门控整合、peer feedback

四、最大的风险点：Model Merging 文献

⚠️ 门控公式 W' = W_A + M⊙(W_B - W_A) 本身不是新东西。

TALL-Mask、Ties-Merging、OBIM 等 Model Merging 论文已经用了这个公式来合并 fine-tuned 模型。

但这不意味着 AMP 被抢占了。

关键差异：

Model MergingAMP
场景集中式，offline，一次性合并去中心化，online，实时交互
数据合并者能看到所有模型的训练数据和验证数据每个 agent 只能看到自己的私有数据
M 的来源算法自动计算（saliency / sign consensus / magnitude）Agent 自主决定——基于 trust + 私有数据验证
双向性单向合并（A 和 B 合并成 C，A 和 B 不更新）双向互学习（A 从 B 学，B 也从 A 学）
反馈无有——告知对方哪些被接受了
论文定位策略：承认门控公式来自 Model Merging 文献（诚实引用），但强调 AMP 的创新在于 M 是怎么来的（agent 自主、trust-driven、基于私有数据验证、双向反馈），而不是公式本身。

五、论文叙事建议

基于这次审查，建议论文把贡献聚焦在以下三个"只有 AMP 做了"的点：

Contribution 1：Agent 自主知识选择（最强）

现行 FL 中，服务器决定怎么聚合。AMP 中，每个 agent 自己决定学不学、学什么、学多少。
最接近的 MOSAIC 只做 RL 策略组合，ATCL 只在服务器端做 trust 管理。

Contribution 2：双向 Feedback 机制（唯一）

没有任何现有工作让模型告知对方"我接受了你的哪些知识"。这个 feedback 通道把单向的"聚合"变成了双向的"对话"。

Contribution 3：基于私有数据验证的 trust（独特）

现有 trust-aware FL 在服务器端用公共指标评估客户端。AMP 的 trust 是每个 agent 在自己的私有数据上验证对方权重后建立的——这更符合"联邦"的精神：没有人能看到所有人的数据，所以信任必须是本地构建的。

叙事框架

"现有 FL 聚合是初等运算（加权平均），AMP 将它升级为高等运算（选择性互学习）。
就像学术界不把所有论文等权平均——学者选择性引用、基于同行评议建立信任、通过论文反馈进行对话。
AMP 把这个知识传播机制形式化为一个数学协议。"

## 二、投稿级别分析

### 顶会可行性

| 会议 | 可行性 | 需要什么 | 阻碍 |
|------|--------|---------|------|
| **ICML** | 低-中 | 收敛理论 + 强实验 | 理论要求高，纯方法论文难中 |
| **NeurIPS** | 中 | 综合实验 + 消融 + 可能理论 | 竞争极其激烈 |
| **ICLR** | 中 | 理论创新突出或实验规模大 | 偏重理论 |
| **AAAI** | 中-高 | 扎实实验 + 清晰贡献陈述 | 投稿量大但接收量大 |
| **IJCAI** | 中-高 | 同上 | 受限于两年一届 |
| **AISTATS** | 中-高 | 理论+实验平衡 | 窗口期合适 |
| **KDD** | 中 | FL 角度要突出 | 偏应用 |

### 现实判断

**AAAI / IJCAI 是最现实的目标。** 理由：
1. 接收量大（AAAI ~23%，IJCAI ~15-20%）
2. 对方法创新 + 扎实实验的论文友好
3. 不一定要求 deep theory（不像 ICML/ICLR）
4. FL 方向的论文在这两个会上有稳定的接收量

**ICML / NeurIPS 是 stretch goal。** 需要额外的理论贡献（收敛性证明）或更大规模的实验。

### 时间线

```
2026.06 — 现在是 6 月 19 日
         AAAI 2027 deadline: 通常在 8 月中旬
         ICLR 2027 deadline: 通常在 9 月底
         → Phase 0 实验 + 写论文 = 6-8 周 → 来得及投 AAAI/ICLR

         ICML 2027 deadline: 通常在 1 月中旬
         → 时间更充裕，可以加上理论工作
```

**建议先投 AAAI 2027（8月 deadline），不中再改投 ICLR 2027（9月 deadline），再不中 ICML 2027（1月 deadline）。**

## 三、对你（学生）的帮助

### 直接收益

| 收益 | 大小 | 说明 |
|------|------|------|
| **毕业** | 🔴 关键 | 如果这是你的学位论文方向，发一篇 AAAI 级别的论文足以满足毕业要求 |
| **学术声誉** | 🟡 中等 | AFP 的方向够新，如果做出来，可能打开一个子领域——"agentic federated learning" |
| **后续研究** | 🟢 高 | 这个方向可以持续做：理论分析、扩展到多模态、真实部署、与其他 FL 方法的理论对比 |
| **工业界关注** | 🟡 中等 | 异构 FL + 隐私 + 安全 是工业界的真实痛点 |

### 风险

| 风险 | 严重性 | 应对 |
|------|--------|------|
| **Phase 0 实验结果负面** | 🔴 高 | 准备好接受——如果 AFP 不比 FedAvg 好，需要理解为什么，这本身可能是理论贡献 |
| **导师不认可新方向** | 🟡 中 | 旧方向的导师反馈已充分纳入 AFP 设计（五连问全部有答案）。但导师可能认为"离题太远" |
| **被 MOSAIC 作者抢先** | 🟡 中 | MOSAIC 已发表（2025.06），但他们做的是 RL。AFP 的攻击点不同（权重空间 + feedback + trust） |
| **Model Merging 社区覆盖** | 🟢 低 | Model Merging 关注的是 offline 合并，不是 online P2P 互学习。不太可能突然转向 |
| **计算资源不足** | 🟡 中 | DGX Spark (121GB) 够跑 Pythia 系列的小模型，但大模型（7B+）跑不了全参数训练 |

### 真正的问题

**这个方向能让你在学术界立足吗？**

能，但有一个前提：**你必须把它做成一个"别人不得不引"的工作。** 

"不得不引"的意思是：
- 你是第一个提出 "Agentic P2P Mutual Learning" 这个概念的
- 后续任何做 P2P FL + agent autonomy 的工作都会引用你
- 或者：你的 feedback 机制被广泛采用为标准协议的一部分

要达到这个目标，AAAI 一篇论文不够——需要后续的理论论文、实验扩展、开源代码（被广泛使用）。

**但 AAAI 一篇论文是起点。没有这个起点，后面都没有。**

## 四、竞争力结论

### 这篇论文的竞争定位

```
        理论深度
          ↑
          |  ICML/ICLR 区      ← AFP 目前不在这个区
          |  (需要收敛性证明)
          |
          |       ● AFP       ← AAAI/IJCAI 区
          |  (方法创新 + 实验)
          |
          |  应用论文区
          |  (纯 PRM 联邦训练)
          └──────────────────→ 实验规模
```

### 与其他学生工作的对比

一个典型的学生论文：
- 拿一个现有方法，在某个新任务上跑实验 → **不引用也没人在意**
- 提出一个小的改进，提升 0.5% → **审稿人可能说 incremental**

AFP：
- 提出了一个新的范式概念（Agentic P2P Mutual Learning）
- 合成的组件来自多个子领域，组合本身是新的
- 有清晰的"为什么比现有方法好"的直觉
- 有可操作的研究路线图（Phase 0→4）

**AFP 比 90% 的学生项目更有潜力。** 但潜力不等于结果——执行决定一切。

### 最终判断

```
问题: 这篇论文级别高吗？
回答: 够 AAAI。冲 ICML 需要理论。冲 NeurIPS 需要大实验。

问题: 对你有帮助吗？
回答: 有。如果你发出来，这可能是你学术生涯的第一个"代表作"。
      但前提是你能把 Phase 0 跑出来，结果正面，论文写好。

问题: 最大的风险是什么？
回答: 不是被 scoop，不是审稿人不喜欢——
      是 Phase 0 跑出来 AFP = FedAvg（没有显著差异）。
      如果这个发生了，你需要一套备用方案：
      → 为什么 AFP = FedAvg？这本身是理论贡献。
      → 在什么条件下 AFP > FedAvg？这引导后续研究。
      → 把 AFP 的失败分析写成论文（审稿人喜欢诚实的负面结果）。
```

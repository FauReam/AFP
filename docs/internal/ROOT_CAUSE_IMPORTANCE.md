# Importance 失效分析（基于实测数据）

> 2026-07-05 | 基于 c1m1, c1m5 实验结果

## 实测数据

### 模型权重偏移

| 模型 | mean ΔW | max ΔW |
|------|---------|--------|
| code_e1 | 0.26% | 6.2% |
| medical_e1 | 1.91% | 13.5% |
| medical_e5 | 1.95% | 14.5% |

### Importance cosine

| 实验 | 模型对 | L2 cosine |
|------|--------|-----------|
| c1m1 | code_e1 + med_e1 | 0.996 |
| c1m5 | code_e1 + med_e5 | 0.948 |

### IVN 结果

| 实验 | FedAvg(α=0.1) | AFP(τ=0.5) | IVN |
|------|---------------|-------------|-----|
| c1m1 | **+0.014** | -0.149 | -0.158 |
| c1m5 | -0.016 | -0.807 | -0.156 |

所有 AFP/IVN 结果为负。IVN 每次 1 轮收敛。

## 根因 #1：门控范围太窄（直接原因）

```
gate[j] = τ / (τ + imp[j])

τ=0.5, imp ∈ [0.1, 1.0] → gate ∈ [0.33, 0.83]
```

每个 block 至少混入 33% 的 peer 权重。但实测只有 ≤10% 混合（FedAvg α=0.1）能保持中性 → 门控下限 0.33 远超可接受上限 0.1。

**这不是 importance 度量的问题——是 gate 函数参数 τ=0.5 不合适。**

## 根因 #2：模型差异太小（根本原因）

1 epoch full-FT 只改变了权重的 0.26-1.95%。99% 的 backbone 仍是原始 Pythia。两个模型本质上是同一个基础模型的微小扰动。在此差异度下：

- **线性插值 loss 是平坦的**（LMC 成立）
- 任何显著混合（>10%）都等于在盆地内随机走
- Gate 无法区分 block 因为 importance pattern 高度相关

## 解决路径

1. **降 τ**：τ=0.1 时 gate ∈ [0.09, 0.5]；τ=0.05 时 gate ∈ [0.05, 0.33]。可以让重要 block 真正被保护。
2. **增加差异度**：更多 epoch、更高 LR、更大数据量。
3. **LMC barrier 测试**：在模型间做线性插值 loss scan，定量验证 barrier 是否存在。
4. **换门控函数**：linear gate `clamp(1-imp/τ, 0, 1)` 可以输出 0（完全保护）。
